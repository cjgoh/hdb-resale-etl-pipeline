HDB Data Platform Deployment Checklist
--------------------------------------

1. Pre-Deployment
- [ ] Confirm AWS account billing alerts are enabled
- [ ] Verify correct AWS region (e.g., ap-southeast-1 for Singapore)
- [ ] Install Terraform and configure AWS CLI with credentials
- [ ] Verify IAM permissions for VPC, S3, Lambda, Glue, Athena, CloudTrail
- [ ] Configure Terraform state backend (S3 + DynamoDB) if team deployment

2. Networking
- [ ] Create VPC (CIDR block e.g. 10.0.0.0/16)
- [ ] Create public subnet (for NAT Gateway)
- [ ] Create private subnet (for Lambda, Glue, S3, Athena)
- [ ] Deploy Internet Gateway and attach to VPC
- [ ] Deploy NAT Gateway in public subnet with Elastic IP
- [ ] Configure route tables (public → IGW, private → NAT)

3. Storage
- [ ] Create S3 buckets: Raw, Cleaned, Transformed, Failed, Hashed
- [ ] Enable S3 encryption (KMS)
- [ ] Configure bucket policies for restricted access
- [ ] Set up partitioning scheme (year/month/town)
- [ ] Enable S3 versioning for disaster recovery
- [ ] Configure lifecycle policies (archive older data to Glacier)

4. Security
- [ ] Create IAM roles for Lambda, Glue, Athena, S3
- [ ] Attach least-privilege policies
- [ ] Enable CloudTrail for logging
- [ ] Configure CloudWatch alarms for job failures
- [ ] Restrict S3 buckets (no public access)
- [ ] Rotate IAM access keys regularly
- [ ] Verify KMS key policies allow only authorized roles
- [ ] Document network segmentation (public vs private)

5. Compute & ETL
- [ ] Deploy Lambda ingestion function (lambda.zip)
- [ ] Test Lambda connectivity to data.gov.sg via NAT Gateway
- [ ] Deploy Glue ETL job with script in S3
- [ ] Create Glue Data Catalog database and tables
- [ ] Document data validation rules (Date, Town, Flat Type, Flat Model, Storey Range)

6. Query & Analytics
- [ ] Deploy Athena workgroup
- [ ] Verify Athena queries run against Transformed S3 data
- [ ] Deploy VPC Endpoint / PrivateLink for Athena → Tableau
- [ ] Configure Tableau to connect via Athena driver
- [ ] Test Tableau extracts for caching and faster dashboards

7. Monitoring & Alerts
- [ ] Set CloudWatch metrics for Lambda duration, errors, throttles
- [ ] Add Glue job success/failure notifications (SNS or email)
- [ ] Enable Athena query logging to CloudWatch
- [ ] Benchmark Glue job execution time and optimize partitioning

8. Validation
- [ ] Run ETL pipeline end-to-end
- [ ] Check Failed dataset for duplicates/anomalies
- [ ] Verify Hashed dataset includes irreversible hashed Resale Identifier
- [ ] Confirm Tableau dashboards load data via Athena
- [ ] Test partitioning improves query performance (with filters)
- [ ] Review CloudTrail logs to ensure all actions are captured

9. Cost Management
- [ ] Set up AWS Budgets and billing alerts
- [ ] Tag all resources (e.g., Project=HDB, Environment=Dev/Prod)
- [ ] Document estimated monthly costs for S3, Glue, Lambda, Athena, Tableau

10. CI/CD & Automation
- [ ] Store Terraform code in Git (with version control)
- [ ] Configure CI/CD pipeline (GitHub Actions, CodePipeline) for automated deployments
- [ ] Add terraform fmt and terraform validate checks in CI

11. Data Governance
- [ ] Define schema evolution strategy in Glue Data Catalog
- [ ] Maintain audit logs of failed records (duplicates, anomalies)
- [ ] Document anomaly detection heuristics and hashing algorithm choice

12. Disaster Recovery
- [ ] Document recovery steps if ETL pipeline fails
- [ ] Test restoring data from S3 versioning or Glacier

13. Logging & Observability
- [ ] Centralize logs (Lambda, Glue, Athena) into CloudWatch Logs
- [ ] Set log retention policies (e.g., 30 days for dev, 1 year for prod)
- [ ] Add dashboards for pipeline health (success/failure counts, ingestion latency)

14. Secrets Management
- [ ] Store API keys or credentials in AWS Secrets Manager or Parameter Store
- [ ] Ensure Lambda functions read secrets securely (no hardcoding)
- [ ] Rotate secrets regularly

15. Compliance & Audit
- [ ] Document data retention policies (raw vs transformed data)
- [ ] Ensure encryption in transit (TLS) and at rest (KMS)
- [ ] Review IAM roles for least-privilege compliance

16. Scaling & Performance
- [ ] Configure Lambda concurrency limits to avoid throttling
- [ ] Tune Glue job worker type and DPUs for large files
- [ ] Use Athena workgroup settings to enforce query limits and cost controls

17. Multi-Environment Setup
- [ ] Define separate workspaces for Dev, Test, and Prod in Terraform
- [ ] Parameterize bucket names, VPC IDs, and resource tags
- [ ] Test pipeline in Dev before promoting to Prod

18. Cleanup & Lifecycle
- [ ] Provide terraform destroy instructions for safe teardown
- [ ] Add lifecycle rules to delete temporary files in Raw zone after X days
- [ ] Archive Cleaned/Transformed data periodically to Glacier

19. Team Handover
- [ ] Provide onboarding guide for new engineers
- [ ] Share architecture diagram and README in repo
- [ ] Document cleanup steps (terraform destroy) to avoid orphaned resources
